一些顺手的命令行工具

发表于2022-11-13

最近关注到 modern-unix 这个项目，在自己的 shell 里试了一下觉得很符合本 noob 的预期。加上最近在各种环境配置终端特别头大，于是简单记一下自己用到的工具。

Before we start

命令行工具的安装通常有多种途径：

  1. 系统自带的包管理器 (apt, homebrew, winget 等) 安装
    • 优点：方便，一行命令就能完成安装；
    • 缺点：一些应用可能未被收录或者过时。
  2. github release 下载
    • 优点：要什么有什么，要哪个版本有哪个版本；
    • 缺点：不一定有相应平台的 prebuilt binary。
  3. 源码编译 (cargo 等)
    • 优点：和包管理器一样方便而且版本更新；
    • 缺点：本地编译耗时可能大于直接下载。

正好之前学习 rust 的时候配置了环境，modern-unix 中大部分工具都可以通过 cargo 安装。

lunarvim

开箱即用的 vim 配置，lunarvim 的安装需要 neovim v0.8.0+。

bat

具有语法高亮的 cat alternative。

lsd

配色很豪华（？）的 ls alternative。

fd

更容易使用且（据说）更快的 find alternative。

ripgrep

更容易使用且（据说）更快的 grep alternative。

zoxide

记录访问路径并进行智能跳转的 cd alternative。

tealdeer

简单的用户手册，输出某个 command 的基本用法。

delta

增强 git diff 的输出结果。
